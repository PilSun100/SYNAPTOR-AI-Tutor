# Demo Scenario

이 문서는 과제 발표 또는 서비스 데모에서 Brain-Sync의 핵심 의도를 보여주기 위한 시연 흐름입니다.

Brain-Sync는 PDF 문제 생성기가 아니라, 업로드 자료를 근거로 사용자의 이해 상태를 추적하고 가장 효율적인 다음 학습 행동을 추천하는 뇌과학 기반 개인화 AI 튜터입니다.

## 준비

백엔드와 프론트엔드를 각각 실행합니다.

```bash
cd backend
source .venv/bin/activate
uvicorn app.main:app --reload
```

```bash
cd frontend
npm run dev
```

시연 주소:

- Web: `http://localhost:5173`
- API Docs: `http://localhost:8000/docs`

## 추천 시연 자료

PDF는 너무 긴 자료보다 1~5페이지 정도의 개념 설명 자료가 좋습니다. 다음 내용이 포함된 자료가 시연에 유리합니다.

- 핵심 개념이 3개 이상 있는 자료
- 원인과 결과 관계가 있는 개념
- 비슷해서 헷갈릴 수 있는 개념
- 예시를 만들 수 있는 개념

## 7분 시연 흐름

### 1. 로그인과 제품 포지셔닝

1. 웹 앱을 열고 회원가입 또는 로그인을 합니다.
2. Dashboard 첫 화면을 보여줍니다.
3. 아직 데이터가 적을 때도 학습 프로필과 추천 학습법 영역이 존재한다는 점을 설명합니다.

발표 포인트:

- 사용자별 학습 데이터가 분리됩니다.
- Brain-Sync는 대화형 챗봇이 아니라 학습 이벤트를 저장하고 개인화하는 서비스입니다.

### 2. PDF 업로드와 RAG 근거 생성

1. Study Room으로 이동합니다.
2. PDF 파일을 업로드합니다.
3. 텍스트 추출 결과와 preview를 보여줍니다.
4. 핵심 개념 추출을 실행합니다.

발표 포인트:

- PDF를 페이지 단위 chunk로 저장합니다.
- 이후 질문, 평가, 힌트는 업로드 자료에서 검색된 evidence chunk를 근거로 사용합니다.
- LLM이 자료 밖 내용을 임의로 만들지 않도록 RAG 구조를 둡니다.

### 3. Active Recall 질문 생성

1. 추출된 개념 중 하나를 선택합니다.
2. 질문 생성을 실행합니다.
3. 질문 카드의 유형, 난이도, 현재 학습 모드를 보여줍니다.

발표 포인트:

- 먼저 설명을 보여주지 않고 질문으로 기억 인출을 유도합니다.
- 질문 유형은 정의형, 원인-결과형, 적용형, 오개념 점검형처럼 학습 목적에 맞게 분리됩니다.

### 4. 답변 평가와 근거 기반 피드백

1. 일부러 불완전한 답변을 입력합니다.
2. 답변 평가 결과를 확인합니다.
3. 정확도, 누락 개념, 오개념 여부, PDF 근거 snippet을 보여줍니다.
4. 상단 개인화 카드의 학습 모드와 권장 난이도가 바뀌는지 확인합니다.

발표 포인트:

- 정답 여부만 판단하지 않고 누락 개념과 오개념을 분리합니다.
- 평가 결과에는 PDF page 기반 evidence가 함께 표시됩니다.
- 사용자의 답변 결과가 개인화 엔진과 학습 프로필에 반영됩니다.

### 5. Hint Ladder

1. Level 1 힌트를 요청합니다.
2. 필요하면 Level 2~3까지 요청합니다.
3. 힌트 ladder가 단계별로 활성화되는 모습을 보여줍니다.

발표 포인트:

- 정답을 바로 알려주지 않고 방향 힌트부터 제공합니다.
- 사용자의 사고 과정을 유지하면서 포기하지 않을 정도의 도움을 제공합니다.
- 힌트 사용량은 힌트 의존도와 개인화 추천에 반영됩니다.

### 6. Self-Explanation

1. 사용자가 자기 언어로 개념을 다시 설명합니다.
2. 정확성, 완전성, 논리 연결성 점수를 확인합니다.
3. 숙련도와 다음 복습 시점이 갱신되는 흐름을 설명합니다.

발표 포인트:

- 맞혔는지보다 자신의 언어로 재구성할 수 있는지를 봅니다.
- 이해한 것 같은 착각을 줄이고 장기 기억화를 돕습니다.

### 7. Report, Daily Review, Dashboard

1. 세션 리포트를 조회합니다.
2. Daily Review 화면으로 이동해 오늘 복습할 개념과 이유를 보여줍니다.
3. Dashboard로 돌아와 학습 프로필, 약한 개념, 오개념 노트, 복습 일정, 최근 세션을 보여줍니다.

발표 포인트:

- 학습 결과가 단발성 점수로 끝나지 않고 다음 복습과 대시보드에 반영됩니다.
- Brain-Sync는 사용자의 약점과 망각 위험을 추적합니다.
- 추천 학습법에는 왜 그 방법을 추천하는지 설명이 함께 제공됩니다.

## 3분 압축 시연 흐름

시간이 부족하면 다음만 보여줍니다.

1. PDF 업로드
2. 개념 추출
3. 질문 생성
4. 일부러 불완전한 답변 제출
5. PDF 근거가 포함된 답변 평가 확인
6. 힌트 ladder 확인
7. Dashboard에서 개인화 추천과 약한 개념 확인

## API 시연 포인트

Swagger 문서에서 다음 API를 보여주면 구조 설명이 쉽습니다.

- `POST /api/materials/upload`
- `POST /api/materials/{material_id}/concepts/extract`
- `POST /api/concepts/{concept_id}/questions/generate`
- `POST /api/questions/{question_id}/answer`
- `POST /api/answers/{answer_id}/hint`
- `POST /api/concepts/{concept_id}/self-explanation`
- `GET /api/profile/learning`
- `GET /api/reviews/daily`
- `GET /api/dashboard/summary`

## 발표 핵심 문장

Brain-Sync는 정답을 알려주는 AI가 아니라, 사용자가 실제로 기억에서 꺼낼 수 있는지 확인하고, 오개념을 근거 기반으로 교정하며, 개인 상태에 맞는 다음 학습법을 추천하는 AI 튜터입니다.

## 예상 질문과 답변

### AI를 직접 학습시킨 프로젝트인가요?

아닙니다. Gemini API를 활용하지만, 프로젝트의 핵심은 모델 학습이 아니라 뇌과학 기반 튜터링 로직, RAG 근거 검색, 개인화 학습 상태 추적, 적응형 난이도 추천을 서비스로 구현한 것입니다.

### 일반 문제 생성기와 무엇이 다른가요?

문제를 만드는 것에서 끝나지 않습니다. 사용자의 답변을 평가하고, PDF 근거를 보여주고, 오개념을 기록하고, 힌트 의존도와 자기 설명 품질을 추적해 다음 학습 방법을 추천합니다.

### 왜 웹 앱과 API를 함께 만들었나요?

웹 앱은 시연성과 사용자 경험을 보여주고, FastAPI는 PDF 처리, RAG, AI 호출, 학습 데이터 저장 구조를 명확히 보여줍니다. 실제 서비스 확장에도 웹 + API 구조가 적합합니다.
